Timeloop Installation Troubleshooting Guide

Environment

  • OS: Debian Linux
  • Compiler: GCC 15
  • Build system: Make + SCons
  • Repo: accelergy-timeloop-infrastructure

Problem 1: Implicit Function Declaration Error in polylib

Error

./applications/testCompressParms.c:58:9: error: implicit declaration of function
'test_Constraints_Remove_parm_eqs'

Cause

Newer GCC versions treat implicit function declarations as errors.
The polylib test file calls undeclared functions.

Fix

Pass warning suppression flags to Make:

rm -rf /tmp/build-timeloop
make install_timeloop CFLAGS="-w -Wno-implicit-function-declaration" CXXFLAGS="-w"

Problem 2: Undefined Reference to glp_* Functions

Error

/usr/bin/x86_64-linux-gnu-ld.bfd: /usr/local/lib/libbarvinok.a(polysign_glpk.o):
in function `solve_lp': undefined reference to `glp_create_prob'

Cause

libbarvinok was compiled with GLPK support but glpk was not linked during the SCons build.
SCons ignores the LDFLAGS environment variable.

Fix

Find the SCons library list:

grep -n "barvinok\|glpk\|LIBS" \
  ~/Documents/fun_stuff/accelergy-timeloop-infrastructure/src/timeloop/src/SConscript

Patch line 69 to add glpk:

sed -i "s/'barvinok', 'isl'/'barvinok', 'glpk', 'isl'/" \
  ~/Documents/fun_stuff/accelergy-timeloop-infrastructure/src/timeloop/src/SConscript

Problem 3: Undefined Reference to lt_dl* and gz* Functions

Error

undefined reference to `lt_dlinit'
undefined reference to `gzopen'

Cause

libglpk statically linked requires libltdl and zlib as transitive dependencies.

Fix

Install the missing dev packages:

sudo apt-get install -y libltdl-dev zlib1g-dev

Patch the SConscript to add ltdl and z:

sed -i "s/'glpk', 'isl'/'glpk', 'ltdl', 'z', 'isl'/" \
  ~/Documents/fun_stuff/accelergy-timeloop-infrastructure/src/timeloop/src/SConscript

Building Timeloop (after all patches)

Skip the full make install_timeloop. Build only the timeloop SCons step:

cd ~/Documents/fun_stuff/accelergy-timeloop-infrastructure/src/timeloop \
  && cp -r pat-public/src/pat src/pat \
  && scons -j8 --with-isl --static --accelergy \
  && scons -j8 --with-isl --accelergy

Installing Binaries

Copy built binaries to ~/.local/bin:

cp build/timeloop-mapper  ~/.local/bin/timeloop-mapper
cp build/timeloop-metrics ~/.local/bin/timeloop-metrics
cp build/timeloop-model   ~/.local/bin/timeloop-model

Ensure ~/.local/bin is in PATH:

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

Problem 4: Missing Shared Libraries at Runtime

Error 1

timeloop-model: error while loading shared libraries:
libntl.so.44: cannot open shared object file: No such file or directory

Fix

NTL was installed to /usr/local/lib but the linker cache was stale:

sudo ldconfig

Error 2

timeloop-model: error while loading shared libraries:
libtimeloop-mapper.so: cannot open shared object file: No such file or directory

Fix

Copy the timeloop shared library to the system lib path:

sudo cp \
  ~/Documents/fun_stuff/accelergy-timeloop-infrastructure/src/timeloop/build/libtimeloop-mapper.so* \
  /usr/local/lib/
sudo ldconfig

Verification

timeloop-model --version

Expected output:

input file: --version
ERROR: Input configuration file does not end with .cfg, .yml, or .yaml

This is normal. Timeloop expects a config file, not a --version flag.
The binary is working correctly.
